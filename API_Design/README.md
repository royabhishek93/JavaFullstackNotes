## Interview Prep Strategy

1. **Start here:** Q35-Q36 (HTTP fundamentals - 6 minutes)
2. **Then:** Q38 (error handling - 3 minutes)
3. **Advanced:** Q37, Q39 (versioning, pagination - 8 minutes)
4. **Security:** Q40 (API security - 4 minutes)
5. **Practice:** Design a RESTful API (1-2 hours)

## Interview Frequency

- **65%** - Q35 HTTP methods and idempotency
- **62%** - Q36 HTTP status codes
- **58%** - Q37 API versioning
- **55%** - Q38 Error responses
- **52%** - Q39 Pagination and filtering
- **50%** - Q40 API security

## Key Topics Covered

- RESTful principles and constraints
- HTTP methods (GET, POST, PUT, PATCH, DELETE)
- Idempotency guarantees
- Status codes (2xx, 4xx, 5xx)
- Versioning (v1/users vs header-based)
- Pagination (offset, cursor)
- Error response formats (RFC 7807 Problem Details)
- API authentication (JWT, OAuth2)
